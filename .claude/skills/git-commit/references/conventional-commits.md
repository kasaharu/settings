# Conventional Commits リファレンス

## フォーマット

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

## タイプ一覧

| タイプ | 説明 | 例 |
|--------|------|-----|
| `feat` | 新機能の追加 | `feat: ユーザー認証機能を追加` |
| `fix` | バグ修正 | `fix: ログイン時のエラーを修正` |
| `docs` | ドキュメントのみの変更 | `docs: READMEを更新` |
| `style` | コードの意味に影響しない変更（空白、フォーマット等） | `style: インデントを修正` |
| `refactor` | バグ修正でも機能追加でもないコード変更 | `refactor: 認証ロジックを整理` |
| `perf` | パフォーマンス改善 | `perf: クエリを最適化` |
| `test` | テストの追加・修正 | `test: ユーザーAPIのテストを追加` |
| `build` | ビルドシステムや外部依存関係の変更 | `build: webpack設定を更新` |
| `ci` | CI設定ファイルやスクリプトの変更 | `ci: GitHub Actionsを追加` |
| `chore` | その他の変更（src/testに影響しない） | `chore: 依存関係を更新` |
| `revert` | 以前のコミットを取り消し | `revert: feat: ユーザー認証機能を追加` |

## スコープ（オプション）

変更の影響範囲を括弧内に記載：
- `feat(auth): ログイン機能を追加`
- `fix(api): エンドポイントのエラーを修正`
- `docs(readme): インストール手順を追加`

## 破壊的変更

破壊的変更（Breaking Change）がある場合：
- タイプの後に `!` を付ける: `feat!: APIのレスポンス形式を変更`
- またはフッターに `BREAKING CHANGE:` を記載

## 良いコミットメッセージの例

```
feat(user): プロフィール編集機能を追加

ユーザーがプロフィール画像と自己紹介を編集できるようになった。
画像は最大2MBまでアップロード可能。

Closes #123
```

```
fix: ログイン時のセッションエラーを修正

セッションの有効期限切れ時に適切にリダイレクトするよう修正。
```
