# スキル作成ガイド

Claude Code のスキル作成を支援します。
新しいスキルを作成したいとき、このガイドに従って対話的にサポートしてください。

## スキル作成ワークフロー

以下の6ステップで進行します：

1. 要件のヒアリング
2. スキル名と説明の決定
3. ディレクトリ構造の設計
4. SKILL.md の作成
5. 補助ファイルの作成（必要な場合）
6. 検証と確認

---

## ステップ1: 要件のヒアリング

ユーザーに以下を確認してください：

1. **スキルの目的**: 何を自動化・支援したいか？
2. **発動条件**: どのような発言・状況でスキルを発動させたいか？
3. **必要なツール**: どのツールを使用するか？（Read, Bash, Grep など）
4. **スコープ**: プロジェクト固有か、個人用（全プロジェクト共通）か？

**質問例**:
```
スキルを作成するにあたり、いくつか確認させてください：

1. このスキルは何をするものですか？
2. どんなときにこのスキルが発動してほしいですか？
3. 特定のプロジェクト専用ですか、それとも全プロジェクトで使いたいですか？
```

---

## ステップ2: スキル名と説明の決定

### 命名規則

| 項目 | ルール |
|------|--------|
| 文字数 | 最大64文字 |
| 使用可能文字 | 小文字、数字、ハイフン（`-`）のみ |
| 予約語 | `anthropic`, `claude` は使用不可 |
| 推奨形式 | 動名詞形（例: `reviewing-code`, `generating-docs`） |

### description の書き方

**重要**: description はスキル発動の判断に使われる最も重要なフィールドです。

**必須要素**:
1. スキルが「何をするか」を具体的に
2. 「いつ発動すべきか」のトリガーキーワード

**良い例**:
```yaml
description: PRのコードレビューを実行し、問題点と改善提案を報告する。ユーザーが「レビューして」「PRを確認して」「コードチェック」と言ったときに発動。
```

**悪い例**:
```yaml
description: コードを見る  # 曖昧すぎて発動条件が不明確
```

詳細は [reference.md](reference.md) を参照。

## ステップ3: ディレクトリ構造の設計

### 保存先の選択

| スコープ | パス | 用途 |
|---------|------|------|
| 個人用 | `~/.claude/skills/{name}/` | 全プロジェクト共通で使う |
| プロジェクト固有 | `.claude/skills/{name}/` | チームで共有、Git管理 |

### 構成パターン

**シンプル構成**（内容が少ない場合）:
```
{name}/
└── SKILL.md
```

**フル構成**（詳細なリファレンスが必要な場合）:
```
{name}/
├── SKILL.md              # メイン（500行以内推奨）
├── reference.md          # 詳細な仕様
├── examples.md           # 使用例
└── scripts/              # 実行スクリプト（任意）
    └── helper.py
```

## ステップ4: SKILL.md の作成

### 基本構造

```yaml
---
name: スキル名
description: スキルの説明。発動条件のキーワードを含める。
allowed-tools: Read, Grep, Glob  # 任意
---

# スキル名

## 概要
[このスキルの目的を1-2文で説明]

## 実行手順

### ステップ1: [アクション名]
[具体的な指示]

### ステップ2: [アクション名]
[具体的な指示]

## 出力フォーマット
[期待する出力形式を示す]

## 注意事項
[エラーハンドリングや制約事項]
```

テンプレートは [templates/SKILL-TEMPLATE.md](templates/SKILL-TEMPLATE.md) を使用してください。

---

## ステップ5: 補助ファイルの作成

SKILL.md が500行を超える場合、または詳細なリファレンスが必要な場合：

| ファイル | 用途 |
|---------|------|
| `reference.md` | API仕様、設定オプションの詳細 |
| `examples.md` | 入出力の具体例 |
| `scripts/` | 実行用スクリプト |

**参照方法**（SKILL.md内）:
```markdown
詳細は [reference.md](reference.md) を参照してください。
```

**注意**: 参照は1階層のみ（A→B→C のような深いネストは避ける）

実例は [examples.md](examples.md) を参照。

## ステップ6: 検証と確認

### チェックリスト

```
YAML frontmatter:
- [ ] name が64文字以内で、小文字・数字・ハイフンのみ
- [ ] description が1024文字以内
- [ ] description にトリガーキーワードを含む
- [ ] --- で正しく囲まれている（1行目から開始）

ファイル構造:
- [ ] SKILL.md がスキルディレクトリ直下にある
- [ ] SKILL.md が500行以内（超える場合は分割）

内容:
- [ ] 手順が明確でステップバイステップ
- [ ] 曖昧な表現がない
- [ ] 出力フォーマットが定義されている
```

### 検証コマンド

```bash
# ディレクトリ構造の確認
ls -la ~/.claude/skills/{スキル名}/

# SKILL.md の行数確認
wc -l ~/.claude/skills/{スキル名}/SKILL.md

# YAML frontmatter の確認
head -10 ~/.claude/skills/{スキル名}/SKILL.md
```
