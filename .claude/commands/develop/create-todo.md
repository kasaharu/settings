# TODO Generator

開発タスクの TODO を作成し、`TODO-*.md` として出力するコマンドです。
この処理のやりとりは日本語でお願いします。

## 入力

- `$ARGUMENTS` にタスクの説明が渡される
- **引数が空の場合**: AskUserQuestion で「どのようなタスクの TODO を作成しますか？」と入力を促す。ユーザーの回答を以降のタスク説明として使う
- **引数がある場合**: そのままタスク説明として使う

## タスク

### ステップ 1: コンテキストの自動収集（質問なし）

以下を並列で収集する:

1. **ブランチ情報**: `git branch --show-current` でブランチ名を取得し、チケット番号（例: `MG-7387`）やタスクタイプ（`feature/` → 新規機能、`fix/` → バグ修正、`refactor/` → リファクタリング）を抽出
2. **プロジェクト情報**: `AGENTS.md` / `CLAUDE.md` / `package.json` 等からプロジェクトの技術スタック・規約を把握
3. **既存 TODO**: カレントディレクトリの `TODO-*.md` を確認し、重複があれば警告

### ステップ 2: コード調査・分析

タスク説明とブランチ情報をもとに、Agent ツール（Explore）を使って関連コードを深く調査する:

- タスク説明に含まれるキーワードで関連ファイルを探索
- 対象ファイルを実際に読み込み、現状の実装を理解する
- 既存のコードパターン・アーキテクチャを把握する
- タスク完了に必要な変更箇所を具体的に特定する

### ステップ 3: TODO の生成・出力

コード調査の結果に基づき、**具体的で実行可能な** TODO を生成してファイルに書き出す。

#### ファイル名

- チケット番号がある場合: `TODO-{チケット番号}.md`
- ない場合: `TODO-{タスクのキーワード}.md`

#### 出力フォーマット

```markdown
# TODO: {タスクの要約}

## 概要

- **タスク**: {タスク説明}
- **チケット**: {チケット番号（あれば）}
- **対象**: {主な対象ファイル・モジュール}

## TODO

### 1. {作業カテゴリ}
- [ ] {具体的なタスク（対象ファイルパスや関数名を含む）}
- [ ] {具体的なタスク}
<!-- commit: "{コミットメッセージ例}" -->

### 2. {作業カテゴリ}
- [ ] {具体的なタスク}
<!-- commit: "{コミットメッセージ例}" -->

## 関連ファイル

- `{ファイルパス}` - {役割の簡潔な説明}

## メモ

{実装上の注意点・参考情報があれば}
```

#### TODO の品質基準

- 各タスクは「何をどのファイルで行うか」が明確であること
- テンプレ的な汎用タスク（「テストを書く」「スタイリング」等）ではなく、コード調査に基づいた具体的なタスクにすること
- タスクの順序は依存関係を考慮すること
- 不要なセクションは省略してよい

#### コミット粒度の指針

- **各ステップ（`### N. {作業カテゴリ}`）が 1 コミットに対応する** ことを意識してステップを分割する
- ステップ内の変更量が大きい場合は、ステップ内をサブステップに分け、サブステップ単位でもコミットできるようにする
- 各ステップの末尾に `<!-- commit -->` マーカーを付け、コミットポイントであることを明示する
- ステップの分割基準: 「このステップだけをコミットしてもビルドが壊れない」状態を保てること

### ステップ 4: 完了報告

作成したファイルのパスと TODO の項目数を報告する。
