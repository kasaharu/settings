# スキル仕様リファレンス

YAML frontmatter と SKILL.md の詳細仕様です。

---

## YAML Frontmatter 仕様

### 必須フィールド

| フィールド | 型 | 制限 | 説明 |
|-----------|------|------|------|
| `name` | string | 最大64文字 | 小文字、数字、ハイフンのみ。ディレクトリ名と一致させる |
| `description` | string | 最大1024文字 | スキルの説明と発動条件。空不可 |

### オプションフィールド

| フィールド | 型 | 説明 |
|-----------|------|------|
| `allowed-tools` | string | カンマ区切りで使用可能ツールを制限 |
| `model` | string | 使用するモデルを指定 |

### YAML 記述ルール

1. **必ず1行目から `---` で開始**（空行を入れない）
2. **`---` で終了してからMarkdown本文**
3. **インデントはスペース**（タブ不可）
4. **特殊文字を含む値はクォートで囲む**

**正しい例**:
```yaml
---
name: my-skill
description: スキルの説明文。「トリガーワード」と言ったときに発動。
allowed-tools: Read, Grep
---

# 本文開始
```

**間違った例**:
```yaml

---
name: my-skill
```
↑ 1行目に空行がある（NG）

## description の書き方

### 基本原則

1. **三人称で書く**（「私」「あなた」は使わない）
2. **具体的なアクションを列挙**
3. **トリガーキーワードを含める**

### パターン別テンプレート

**パターン1: アクション列挙型**
```yaml
description: [アクション1]、[アクション2]、[アクション3]を行う。ユーザーが「[キーワード1]」「[キーワード2]」と言ったときに発動。
```

**パターン2: 条件明示型**
```yaml
description: [何をするか]。[特定のファイル形式]を扱うとき、または[特定の状況]で発動。
```

**パターン3: ドメイン特化型**
```yaml
description: [ドメイン]に関する[タスク]を実行。[関連キーワード1]、[関連キーワード2]に対応。
```

## allowed-tools の指定

### 利用可能なツール一覧

| ツール名 | 用途 |
|---------|------|
| `Read` | ファイル読み込み |
| `Grep` | ファイル内検索 |
| `Glob` | ファイルパターンマッチ |
| `Bash` | コマンド実行 |
| `Edit` | ファイル編集 |
| `Write` | ファイル書き込み |
| `WebFetch` | Web コンテンツ取得 |
| `WebSearch` | Web 検索 |

### 制限の例

```yaml
# 読み取り専用スキル
allowed-tools: Read, Grep, Glob

# ファイル操作可能
allowed-tools: Read, Grep, Glob, Edit, Write

# Bash でPythonスクリプトのみ許可
allowed-tools: Read, Bash(python:*)
```

## ディレクトリ構造

### 優先順位（高い順）

1. **Enterprise**: 組織全体（Platform設定）
2. **Personal**: `~/.claude/skills/` - 個人用、全プロジェクト共通
3. **Project**: `.claude/skills/` - プロジェクト固有、Git管理可能
4. **Plugin**: プラグイン内の `skills/` ディレクトリ

### ファイル命名規則

| ファイル | 規則 |
|---------|------|
| `SKILL.md` | 必須、大文字 |
| `reference.md` | 小文字推奨 |
| `examples.md` | 小文字推奨 |
| `scripts/` | ディレクトリ名は小文字 |

### パスの書き方

- **常にフォワードスラッシュ**を使用: `scripts/helper.py`
- **Windows形式は使わない**: `scripts\helper.py` （NG）

## トラブルシューティング

| 症状 | 原因 | 対策 |
|------|------|------|
| スキルが発動しない | description が曖昧 | トリガーキーワードを追加 |
| スキルが読み込まれない | パスが間違っている | `~/.claude/skills/{name}/SKILL.md` を確認 |
| YAMLエラー | 構文ミス | 1行目に `---` があるか、インデントがスペースか確認 |
| 参照ファイルが読まれない | ネストが深い | 参照は1階層のみに |
| 複数スキルが競合 | description が似ている | トリガーキーワードを差別化 |

### デバッグ方法

```bash
# デバッグモードで起動
claude --debug

# スキルディレクトリの確認
ls -la ~/.claude/skills/

# YAML構文チェック（yqが必要）
yq e '.' ~/.claude/skills/{name}/SKILL.md
```

## スキル vs コマンド vs その他

| 機能 | 実行方法 | 用途 |
|------|---------|------|
| **Skills** | 自動発動 | 専門知識をClaudeに付与 |
| **Commands** | `/command` で明示的に実行 | 再利用可能なプロンプト |
| **CLAUDE.md** | 毎回自動読込 | プロジェクト全体の指示 |
| **Hooks** | ツールイベント時 | スクリプト実行 |
| **MCP servers** | Claudeが必要時に呼出 | 外部ツール連携 |
