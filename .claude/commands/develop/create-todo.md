# Frontend TODO Generator

フロントエンド開発用の TODO を整理し、TODO-xxx.md として出力するコマンドです。
この処理のやりとりは日本語でお願いします。

## タスク

### ステップ 1: 開発内容のヒアリング

ユーザーに以下の情報を確認してください:

1. **開発する機能・画面の概要**
   - 何を作るのか（例: ログイン画面、ダッシュボード、設定ページ）
   - 主な目的・ユースケース

2. **技術スタック**（わかる範囲で）
   - フレームワーク（React, Vue, Next.js など）
   - 状態管理（Redux, Zustand, Jotai など）
   - スタイリング（CSS Modules, Tailwind, styled-components など）

3. **要件・仕様**
   - 必要な機能のリスト
   - デザインカンプや参考資料があれば確認

### ステップ 2: TODO の整理

ヒアリング内容をもとに、以下のカテゴリで TODO を整理:

```markdown
## 概要
- 機能名:
- 目的:
- 技術スタック:

## TODO

### 1. 環境・セットアップ
- [ ] 必要なパッケージのインストール
- [ ] ディレクトリ構成の作成
- [ ] 設定ファイルの準備

### 2. コンポーネント実装
- [ ] コンポーネント名1
- [ ] コンポーネント名2
...

### 3. 状態管理・データ取得
- [ ] Store/State の設計
- [ ] API 連携
...

### 4. スタイリング
- [ ] レイアウト
- [ ] レスポンシブ対応
...

### 5. テスト
- [ ] ユニットテスト
- [ ] E2E テスト
...

### 6. その他
- [ ] アクセシビリティ対応
- [ ] パフォーマンス最適化
...
```

### ステップ 3: ファイル名の確認

AskUserQuestion ツールを使用して、出力ファイル名を確認:

- 「TODO-xxx.md の xxx 部分を入力してください（例: auth, dashboard, settings）」
- ユーザーが入力した値を使って `TODO-{入力値}.md` というファイル名にする

### ステップ 4: ファイル出力

カレントディレクトリに `TODO-{入力値}.md` を作成し、整理した TODO を出力。

出力後、以下を報告:
- 作成したファイルのパス
- TODO の項目数
- 次のアクションの提案

## 実行例

```
/create-todo
```

これにより:
1. 開発内容をヒアリング
2. TODO を整理・構造化
3. ファイル名を確認
4. TODO-xxx.md として出力
